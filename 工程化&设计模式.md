**webpack 构建过程**  
 读取配置 → 编译模块 → 生成依赖图 → 打包输出

**loader 和 plugin 区别**  
Loader = 文件转换器，把非 JS 文件（CSS/图片等）转成 JS 能处理的模块  
Plugin = Webpack 的功能扩展器，在构建过程的不同阶段插入自定义逻辑（生成 html 文件，压缩代码，打包分析等）

**webpack 优化**  
代码分割，tree-shaking，缓存，懒加载

**vite特点**  
启动快：开发环境不打包，利用原生 esmodule 按需编译文件  
开发环境 esbuild 构建，生产环境 rollup 构建

接口易用、易理解、内部高内聚、外部低耦合、可扩展、可测试、文档完善

**webpack常用配置项**

1. entry：入口文件，webpack 开始打包的文件
2. output：输出配置，指定打包后的文件路径和文件名
3. module：模块配置，定义如何处理不同类型的文件，如加载器（loader）
4. plugins：插件配置，添加额外的功能，如代码压缩、HTML 生成等
5. mode：模式配置，指定是开发模式还是生产模式，影响打包行为
6. resolve：模块解析配置，指定模块查找路径和文件扩展名
7. devServer：开发服务器配置，设置本地开发服务器的选项，如端口号、热更新等功能
8. optimization：优化配置，控制代码分割、压缩等优化行为
